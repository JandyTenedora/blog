Sydney-raised, stats and CS graduate from the University of Sydney, now a senior data engineer living in the UK.

This blog documents a return to the core ideas I surrounded myself in when first discovering computer science at university. The exact first principles I long to return to: how computers actually work at the level beneath the frameworks, how data moves across networks, how programs become processes, how the machine computes.

I had never envisioned myself becoming a software or data professional. In truth I had, at university, meandered from a fascination with economics and econometrics to statistics, to finally deciding I may as well pick up some coding electives on the side. Lo and behold, I find myself now six years into a career as a backend and data engineer. Perhaps I'll write about how that happened one day.

But for now, I want to walk away from the temptation we all have to lose sight of what happens underneath the hood, to spend years operating only at the level of abstraction our jobs let us get away with. This is the deliberate step back down to where the real understanding starts. This is a deliberate step into the kind of depth that I truly think sets people apart, the kind that lets you reason about problems from the ground up and solve things you otherwise simply couldn't.

## What I write about

- **deep-systems projects**: HTTP servers, relational databases, message queues, distributed brokers, wide-column stores. Each one built twice: Java prototype first, then Go. Built to understand the layers of infrastructure I use but haven't had to implement.
- **First-principles reading**: notes from CS:APP, Kurose, SICP. The books that most working engineers skip.
- **Data engineering**: pipelines, orchestration, warehousing, stream processing.
- **German**: I'm learning German. Occasionally I'll write about that too.

## The setup

The underlying system is an Obsidian vault running as a personal learning OS, with Claude Code handling daily planning and note generation. I wrote about how it works in the [first post](/learning-os/building-my-learning-os/).
