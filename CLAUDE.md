# CLAUDE.md

This is a Hugo static site blog deployed to GitHub Pages. It is the public-facing output of Jandy Tenedora's learning-in-public project — separate from a private Obsidian vault that serves as the working knowledge base.

## Project overview

- **Framework**: Hugo (extended v0.157.0) with PaperMod theme (git submodule)
- **Deployment**: GitHub Pages via GitHub Actions — push to `main` triggers automatic deploy
- **Live URL**: https://jandytenedora.github.io/
- **Author**: Jandy Tenedora — data engineer in the UK, upskilling to senior level

## Directory structure

```
content/
  posts/       # blog posts (Markdown)
  about.md     # about page
archetypes/
  default.md   # template for new content
themes/
  PaperMod/    # git submodule — do not edit directly
public/        # build output — do not edit manually, generated by hugo
assets/        # custom CSS/JS overrides (currently empty)
layouts/       # custom layout overrides (currently empty)
static/        # static assets (images, etc.)
hugo.toml      # site configuration
.github/
  workflows/
    hugo.yml   # CI/CD deploy pipeline
```

## Common commands

```bash
# Run local dev server (includes drafts)
hugo server -D

# Build for production
hugo --minify

# Create a new post
hugo new content/posts/post-name.md
```

## Post front matter

Posts use TOML or YAML front matter. Standard fields:

```yaml
---
title: "Post title"
date: 2026-03-07
draft: false
tags: ["tag1", "tag2"]
description: "One-sentence summary shown in post listings."
showToc: true
---
```

- Set `draft: true` while writing; set to `false` when ready to publish
- Tags used so far: `meta`, `obsidian`, `claude-code`, `learning`, `data-engineering`

## Deployment

Pushing to `main` automatically triggers the GitHub Actions workflow (`.github/workflows/hugo.yml`), which builds with Hugo extended and deploys to GitHub Pages. No manual deploy step needed.

## Theme

PaperMod is installed as a git submodule at `themes/PaperMod`. To update it:

```bash
git submodule update --remote themes/PaperMod
```

Do not edit files inside `themes/PaperMod/` directly. Use `assets/`, `layouts/`, or `static/` at the repo root to override theme files.

## What not to touch

- `public/` — generated build output, rebuilt on every deploy
- `themes/PaperMod/` — upstream submodule, customise via root-level overrides
- `.hugo_build.lock` — Hugo lock file, ignore
